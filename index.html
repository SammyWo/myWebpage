Project Idea

Overview

Noise Induced Hearing Loss (NIHL) is the greatest contributor of premature hearing loss in young adults. Johnston's findings based on National Institute for Safety & Health (NIOSH) criteria indicate limiting how long you are exposed to excessive noise above 85dBA can significantly reduce the risk of earring loss (Johnston, 2020). The intended project proposed is a browser-based noise exposure indicator using generic consumer devices such as smartphones or personal computers. Users will be able to open a webpage and using their devices microphone, measure the sound pressure level (SPL) of their environment. If the user is exposed to dangerous levels of SPL over a period of time the program will issue a warning that the user needs to have a rest and reduce noise exposure.

Motivation

Despite strict OH&S regulations introducing noise exposure measures to industrial worksites, other work environments such as offices, loud bars and restaurants still pose a potential risk. Safework Australia recommends that noise exceeding 50dBa for focused conversational work and 70dBA for routine work should be met with some noise exposure procedures (Noise, 2020). By creating a browser-based noise exposure meter, small businesses and individuals will have a cost-effective way to measure the relative loudness of their environment. This will result in more people being aware of their noise exposure and hopefully reduce rates of premature hearing loss.

Description

The Product is intended to be highly accessible. For this purpose, the product will be available using any web browser and compatible with a variety of devices. The most common being a smartphone or tablet. When the user accesses the webpage on their browser and allows access to their device’s microphone, the browser will collect measurements and data from the microphone and display results on the web page. The webpage will display the current SPL in decibels (dBA) and indicate how much excessive noise the user has been exposed to. This can either be displayed as a percentage with 100% being the maximum permitted amount of exposure, or a timer that counts down.

The product will utilise a timer to access how much noise the user has been exposed to. When noise exceeds a threshold, the timer counts down. Once the timer reaches zero, the user will be warned that they have been exposed to excessive noise. Based on the NIOSH recommend exposure limit (Johnston, 2020), the threshold will start at 85dBA with 8 hours of permitted exposure. If the SPL increases by 3dB, the timer will double in speed halving the permitted exposure time. For example; when the user initiates the program, the timer is set to 8 hours. The timer only starts to count down when SPL exceeds 85dBA. If SPL exceeds 88dBA the timer will double in speed until the SPL lowers below 88dBA where it returns to normal speed. If the noise continues to drop below 85dBA the timer will pause.

SPL (dBA) 85 88 91 94 100
NIOSH Permitted noise exposure (Hours) 8 4 2 1 0.25

The product is not intended to record accurate measurements as that would require professional calibrated equipment. This is a tool to keep the user aware of how much noise they are exposed to and only approximate measurements are necessary. The webpage will detail some instructions to the user on setting up their device to capture the best results possible. The above table will be displayed for reference.

Similar tools do exist, such as Audio Tool's Dosimeter which is a paid add-on for their iPhone application (AudioTools - dB, Sound & Audio, 2020). However, there is no such browser-based tool that is easily accessible. There are some browser-based sound level meters such as the Youlean's free loudness meter (Free Online Loudness and Sound Meter by Youlean, 2020). This does not however indicate to the user the amount of noise they are exposed to over a period of time and the average user may not be able to interpret the data presented in an informed manner. The proposed product is intended to be simple so that the results are easily understood.

Tools and Technologies

The product requires access to a microphone. Most consumer grade devices such as smartphones and tablets have the ability to capture an audio source. For more comprehensive setups, use of a measurement microphone and an audio interface can be used for more accurate results.

Inspecting the code of Youlean's Free Loudness Meter, it is possible to create a sound meter on a webpage using html however the sound meter itself is most likely constructed using another programming language. Wilson (2017) developed a simple volume meter to monitor a live audio input using java script and a similar volume meter can be constructed based on Wilson's work. Similar sound meters have also been developed using Python and other programming languages. Using GitHub, the project can be constructed and tested by developers allowing version control and frequent updates.

Skills Required

To develop this product, skills in four different areas are required. First is a basic knowledge of audio and acoustics and how to interpret audio data and provide it to the user in an easily understandable manner. Secondly some aptitude in html is required to structure the webpage. Thirdly, in order to construct the sound level meter and timer, some understanding of a programming language such as java script is required. Last, skills in front end design to create the user interface are required.

Outcome

If the project is successful, there will be an increase in awareness of excessive noise exposure and a reduction in noise induced hearing loss. Because the final product is intended to be free and highly accessible, there is no barrier for the average user to utilise the product. If the initial product is successful, future versions of the software can have customisable features to tailor to different standards and work environments. This will ensure the product is future proof and can adapt to changes in safety standards and practices as they develop.
